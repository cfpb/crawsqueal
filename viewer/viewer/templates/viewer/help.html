Common crawler searches

The crawler stores a nightly snapshot of webpage data that you can search for all sorts of useful things.

You can search for single terms or exact phrases. Examples:

Title

You want a list of all pages with the word "Chopra" in the title.

Search "Chopra"

URL

Find all instances of a string in the URL of a published cf.gov page.

Your stakeholder wants a list of all the pages in a given site section.

Search "/about-us/the-bureau/bureau-structure/"

Components

Search by class names of our design components. Partial searches also work: searching for "notification" will return pages containing "m-notification", "m-notification__borderless", etc.

You're doing an audit of all occurrences of the notification pattern.

Search "m-notification"

Links

Find all instances of a string in the URLs of links listed on a page, excluding the header and footer.

You're looking for all instances of external links to TBD.

Search "TBD"

You want to find and remediate accidental links to the content server.

Search "content.consumerfinance.gov"

You want to find published cf.gov pages with broken links.

Search "none"

You're looking for incorrectly formatted document links.

Search "www.consumerfinance.gov/documents"

Full text

Find all instances of a term or exact phrase in the full text of the <body> of a page, excluding the header and footer.

You're looking for specific phrasing or language that might need to be updated because of a policy change.

Search "medical debt"

HTML

You want to find and remediate manually-added line breaks.

Search "<br>"
